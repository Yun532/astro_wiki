# 02 观测量与参考系变换

本章把上一章的 Doppler factor 和 arrival time 关系转化为 GRB 论文中最常见的观测量语言：flux、fluence、luminosity、isotropic-equivalent energy、rest-frame time / energy band 和 K-correction。后续比较 GRB 030329、GRB 080319B、GRB 221009A 等事件时，很多 apparent contradiction 实际来自能段、时间积分区间、参考系和宇宙学参数不一致。

## 1. 三个参考系层级

GRB 余辉和瞬时辐射文献中至少有三个常用参考系：

1. **Comoving frame**：随 emitting plasma 一起运动的局域参考系，常用 primed quantities，如 `ν'`、`t'`、`I'_{ν'}`。
2. **Source rest frame / cosmological rest frame**：扣除 redshift 后的宿主星系静止系，常用 `t_src = t_obs/(1+z)` 和 `E_src = (1+z)E_obs`。
3. **Observer frame**：探测器实际记录的时间、能段、flux、fluence 和 count rate。

上一章中的 Doppler factor `δ` 连接 comoving frame 与 source-frame line of sight；cosmological redshift `(1+z)` 再把 source-frame photon 映射到 observer frame。

## 2. 时间与频率 / 能量变换

若忽略局域 Doppler factor，只做 cosmological rest-frame 修正，则

```tex
t_{\rm src} = \frac{t_{\rm obs}}{1+z},
```

```tex
\nu_{\rm src} = (1+z)\nu_{\rm obs},
```

```tex
E_{\rm src} = (1+z)E_{\rm obs}.
```

若同时保留 emitting plasma 的 Doppler factor，则

```tex
\nu_{\rm obs} = \frac{\delta}{1+z}\nu',
```

```tex
dt_{\rm obs} = \frac{1+z}{\delta}dt'.
```

这里 `E = hν`，所以 photon energy 与 frequency 使用同样的变换。对 prompt emission，论文常把 spectral peak 写作 observer-frame `E_{p,obs}` 或 rest-frame `E_{p,z}`：

```tex
E_{p,z} = (1+z)E_{p,obs}.
```

使用 Amati / Yonetoku 类关系或跨 source 比较 peak energy 时，应确认使用的是 observer frame 还是 rest frame。

## 3. Flux、luminosity 与 luminosity distance

Bolometric energy flux `F` 是 observer frame 中单位面积、单位时间接收到的能量。若 source bolometric luminosity 为 `L`，luminosity distance 为 `D_L`，定义关系为

```tex
F = \frac{L}{4\pi D_L^2}.
```

因此

```tex
L = 4\pi D_L^2 F.
```

`D_L` 已经包含宇宙学 redshift 对 photon energy、arrival rate 和几何距离的影响；不要在 bolometric luminosity 公式里额外重复乘 `(1+z)`。但如果 `F` 只在有限 instrument band 中测量，就需要 K-correction 才能得到指定 rest-frame band 的 luminosity。

对 spectral flux density，常见定义为

```tex
F_\nu \equiv \frac{dF}{d\nu}.
```

若 source-frame spectral luminosity 为 `L_{\nu_{\rm src}}`，则

```tex
F_{\nu_{\rm obs}} = \frac{1}{4\pi D_L^2}\frac{L_{\nu_{\rm src}}}{1+z},
```

其中

```tex
\nu_{\rm src} = (1+z)\nu_{\rm obs}.
```

等价地，

```tex
L_{\nu_{\rm src}} = 4\pi D_L^2(1+z)F_{\nu_{\rm obs}}.
```

这个 `(1+z)` 的位置经常造成混淆；关键是区分 bolometric flux 和 per-frequency spectral flux density。

## 4. Fluence 与 isotropic-equivalent energy

GRB prompt 论文常报告 fluence：

```tex
S = \int F(t)\,dt,
```

单位通常为 `erg cm^{-2}`。若该 fluence 是在某个 observer-frame band 中测得，并经过 K-correction 转换到指定 source-frame bolometric band，则 isotropic-equivalent radiated energy 为

```tex
E_{\rm iso} = \frac{4\pi D_L^2}{1+z} S_{\rm bolo}.
```

这里除以 `(1+z)` 是因为 fluence 已经对 observer-frame time 积分，source-frame emitted energy 比 observer-frame received photon energy accounting 需要 redshift correction。若论文直接给出 `E_{\gamma,\rm iso}` 或 `E_{\rm iso}`，必须记录：

- observer-frame detector band；
- 转换到的 rest-frame energy band；
- 使用的 spectral model；
- integration time interval；
- cosmological parameters。

例如 GRB 221009A 的 Konus-Wind、Fermi-GBM、INTEGRAL 等 source 对 `E_iso` 或 fluence 的能段和时间积分定义并不完全相同，不能只比较数值。

## 5. Peak luminosity 与 isotropic-equivalent luminosity

若在某个 observer-frame time bin 中测得 bolometric peak flux `P_{\rm bolo}`，常定义 isotropic-equivalent peak luminosity：

```tex
L_{\rm iso} = 4\pi D_L^2 P_{\rm bolo}.
```

这里 `P_{\rm bolo}` 的时间 bin 通常是 observer frame 的 `1 s`、`64 ms` 或 source-specific interval。跨 source 或跨仪器比较 `L_iso` 时，需要检查 time bin 是否已转换到 source frame，以及是否使用相同 bolometric correction。

## 6. Specific intensity invariant

辐射转移中常用的 Lorentz invariant 是

```tex
\frac{I_\nu}{\nu^3} = \frac{I'_{\nu'}}{\nu'^3}.
```

因此若只考虑局域 Doppler transformation，

```tex
I_\nu = \delta^3 I'_{\nu'}.
```

对 unresolved relativistic source，observed flux 还涉及 apparent solid angle、arrival-time compression 和 equal-arrival-time surface 积分。很多 GRB 论文不会从 `I_ν/ν^3` 开始写推导，但 structured jet、off-axis afterglow 和 prompt observables 的几何积分都隐含这一不变量。

## 7. K-correction 的含义

仪器测到的是 observer-frame band `[E_{1,obs}, E_{2,obs}]` 中的 fluence 或 flux。若要比较不同 redshift GRB 在同一个 rest-frame band `[E_{1,src}, E_{2,src}]` 中的能量，需要用 spectral model 做 bandpass correction。

概念上，若 photon spectrum 或 energy spectrum 已知，K-correction 是把 observed band integral 换算到指定 rest-frame band integral 的比例因子。其细节依赖 spectrum convention，例如 photon number spectrum `N(E)` 还是 energy flux spectrum `E N(E)`。因此知识库中记录 `E_iso` 时优先保留原文 source 的 band definition，而不是强行重算。

## 8. 常见误区

- **把 detector band 当成 rest-frame band**：例如 observed `20 keV–10 MeV` 对 `z≈1` source 来说对应 source-frame `40 keV–20 MeV`。
- **比较不同论文的 `E_iso` 不看能段**：GRB 221009A 的不同仪器 source 给出的高能端、低能端和积分时间不同。
- **bolometric 与 band-limited 混用**：`4πD_L²F` 只有在 flux 对目标能段定义清楚时才可比较。
- **忽略 time bin**：peak luminosity 对 binning 敏感，特别是 prompt emission。
- **把 `E_iso` 当成真实能量**：`E_iso` 是假设各向同性发射的等效量；true energy 还需要 jet geometry / beaming correction。

## 9. 和事件页面的接口

- GRB 221009A：prompt energetics 页面中不同仪器的 `E_iso`、fluence、energy band 应保留 source-specific definition。
- GRB 080319B：prompt optical 与 gamma-ray spectral mismatch 比较时，要确认 optical flux 和 gamma-ray spectrum 外推是否在一致的时间段与谱定义下。
- GRB 030329：radio calorimetry 得到的是 kinetic energy / beaming-corrected energy 语境，不应和 prompt `E_iso` 直接等同。
